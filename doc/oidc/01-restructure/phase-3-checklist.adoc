This is the committed, self-detecting checklist for Phase 3 of the Token-Sheriff
rename. Each row carries a read-only *Detect* command (the source of truth — run
it to learn the real state) and an idempotent *Action*. Trust *Detect*, not the
*State* column. (State column last reconciled 2026-07-04 against the
xref:README.adoc#_verification_checklist[Plan 01 verification]; only the OpenSSF
row remains open.)

[NOTE]
====
*Resuming in a fresh checkout/session?* You are in the renamed clone (e.g.
`/Users/oliver/git/TokenSheriff`). Run every *Detect* top-to-bottom to rebuild
the true state, then execute the *Action* of each not-yet-done row. The repo
rename keeps the old URL redirecting, so the primary path needs no re-clone —
`git remote set-url` suffices (row 2).

== Verification (run after all rows DONE)

[source,bash]
----
# repo renamed + redirect works
gh repo view cuioss/TokenSheriff --json name -q .name        # TokenSheriff
git remote get-url origin                                     # ...TokenSheriff.git
# every README badge/link resolves (expect no non-200)
grep -oE 'https?://[^[:space:]"<>)]+' README.md | sort -u | while read -r u; do
  code=$(curl -s -o /dev/null -w '%{http_code}' "$u"); [ "$code" = 200 ] || echo "$code  $u"; done
