= Code Findings: Compatibility vs. Correct Design
:toc: left
:toclevels: 3

== Overview

Semantic analysis of all production code asking one question per artifact:
_"Does this exist because it is the right design, or because we are afraid to break something?"_

Maturity: **pre-1.0** — Breaking changes are free. No deprecation cycles needed. Recommend removing/fixing everything.

Date: 2026-04-01

Modules analyzed:

* `oauth-sheriff-core` (~95 production source files)
* `oauth-sheriff-quarkus` (~35 production source files)
* `oauth-sheriff-quarkus-deployment` (1 file)

== Critical (Security Impact)

No critical findings. The validation pipeline has strong security fundamentals:

* "none" algorithm explicitly rejected via hardcoded `REJECTED_ALGORITHMS` list
* HMAC algorithms (HS256/HS384/HS512) blocked, preventing key confusion attacks
* Embedded JWK attack (CVE-2018-0114) prevented in `TokenHeaderValidator`
* Token size limits enforced at multiple layers
* DPoP replay protection with jti tracking
* All validators throw `TokenValidationException` on failure — no silent pass-through

== High (API / Design)

[cols="1,3,4,3"]
|===
|ID |Location |Issue |Recommendation

|H-1
|`JWKSKeyLoader:333-344`
|**TOCTOU race in initialization.** `initJWKSLoader` checks `if (!initialized)` without synchronization. Two concurrent callers can both enter the block, causing double initialization and potential corruption of `keyInfoMap`. Compare with `HttpJwksLoader.initJWKSLoader` which correctly uses `AtomicReference.compareAndSet`.
|Use `AtomicBoolean.compareAndSet(false, true)` or a `synchronized` block to make initialization idempotent.

|H-2
|`HttpJwksLoader:306-336`
|**Background refresh silent death.** The scheduled refresh task catches only `IllegalArgumentException` and `IllegalStateException`. Any other unchecked exception (NPE, OOME) causes `ScheduledExecutorService` to silently cancel the task permanently — no log, no retry, keys go stale.
|Add a catch-all `catch (Exception e)` that logs WARN and does NOT rethrow, ensuring the scheduled task stays alive.

|H-3
|`DecodedJwt:83-93`
|**Dual accessors on record.** `DecodedJwt` is a `record` with automatic accessors `header()`, `body()`, but also declares manual `getHeader()`, `getBody()`. Callers use both styles inconsistently. Confusing, duplicated API surface.
|Remove `getHeader()` and `getBody()`. Migrate callers to record accessors `header()` and `body()`.

|H-4
|`BearerTokenInterceptor:159-168`
|**@BearerAuth merging documented but not implemented.** `BearerAuth` Javadoc states method-level annotation "adds additional requirements" to class-level. But `extractAnnotation()` returns only the first match — method-level **replaces** class-level. A class `@BearerAuth(requiredRoles={"admin"})` with method `@BearerAuth(requiredScopes={"write"})` enforces only the scope, not the role.
|Either implement annotation merging (safer for a security library) or fix the Javadoc to state method-level replaces class-level.
|===

== Medium (Dead Code / Unnecessary API Surface)

[cols="1,3,4,3"]
|===
|ID |Location |Issue |Recommendation

|M-1
|`BaseTokenContent:75-76`
|**Dead `rawPayload` field.** `getRawPayload()` has zero callers anywhere in the codebase. The `MapRepresentation` lives in memory alongside the already-processed `claims` map, doubling per-token memory.
|Remove `rawPayload` from `BaseTokenContent` and corresponding constructor parameters.

|M-2
|`BaseTokenContent:53`, `AccessTokenContent:65`, `IdTokenContent:65`
|**Unused `@SuperBuilder`.** No code calls `AccessTokenContent.builder()` or `IdTokenContent.builder()`. All instantiation uses explicit constructors. Dead generated code inflates API surface.
|Remove `@SuperBuilder` from all three classes.

|M-3
|`ClaimName:185`
|**Unbounded cache with attacker-controlled keys.** `CLAIM_NAME_CACHE` is an unbounded `ConcurrentHashMap` that caches lookups including misses. Claim names in tokens are attacker-controlled — many unique fake claim names grow this cache without bound. Impact is limited by `ParserConfig` token size limits, but still worth addressing.
|Only cache hits (known `ClaimName` values) or pre-populate from `values()` at class-load and skip caching misses.

|M-4
|`AccessTokenCache:161,220`
|**Weak cache key via `hashCode()`.** Uses `tokenString.hashCode()` (32-bit) as `Integer` key. Realistic collision probability at 1000 entries. Collisions cause permanent cache eviction for affected tokens. Code documents this as intentional trade-off.
|Use full token string or a cryptographic hash (SHA-256 truncated) as cache key.

|M-5
|`JweDecryptor:62`
|**Unhardened JSON parser for JWE.** Creates a standalone `DslJson` instance that bypasses the security-hardened `ParserConfig` (max payload size limits). Attacker-controlled `epk` field in JWE header is parsed without size checks.
|Pass the configured `DslJson` instance through to `JweDecryptor`, or add a size check on the `epk` JSON before parsing.

|M-6
|`JweDecryptor:378-421`
|**Duplicated EC key parsing.** `parseEcPublicKeyFromJwk` duplicates the EC curve mapping and key construction from `JwkKeyHandler.parseEcKey`. Both have their own `getEcParameterSpec()` method. Security fixes to EC key parsing must be applied in two places.
|Reuse `JwkKeyHandler.parseEcKey` by deserializing `epk` JSON to a `JwkKey` record first.

|M-7
|`RefreshTokenValidationPipeline:84-109`
|**Unvalidated token accepted silently.** Refresh tokens bypass all validation (by design for opaque tokens). Any arbitrary string is accepted and wrapped into `RefreshTokenContent`. The container's name does not signal this is unvalidated.
|Rename to `UnvalidatedRefreshToken` or add a `validated=false` flag to prevent consumers from trusting claims.

|M-8
|`IssuerConfig:195-231`
|**Token type validation disabled by default.** `expectedTokenType` defaults to `null`, enabling token type confusion (ID token presented as access token). Documented as intentional due to poor IdP compliance with RFC 9068.
|Log a one-time startup warning when `expectedTokenType` is null indicating token type confusion protection is disabled.

|M-9
|`WellKnownResult:56-95`
|**Dead fields.** `authorizationEndpoint`, `tokenEndpoint`, `userinfoEndpoint` and their accessors have zero callers in production code. Dead API surface.
|Remove the three unused fields and accessors.

|M-10
|`JwtHeader:79-87`
|**Dead record components.** Six fields (`jku`, `x5u`, `x5c`, `x5t`, `x5tS256`, `crit`) are stored but never accessed in production. DSL-JSON silently ignores unknown fields, so removal is safe.
|Remove the six unused fields from the record.

|M-11
|`JWTValidationLogMessages:75-450`
|**12 unused LogRecord constants.** `ERROR.UNSUPPORTED_JWKS_TYPE` (204), `ERROR.CACHE_TOKEN_NO_EXPIRATION` (206-209), `INFO.JWKS_HTTP_LOADED` (3), `INFO.JWKS_BACKGROUND_REFRESH_UPDATED` (5), `WARN.BACKGROUND_REFRESH_SKIPPED` (124), `WARN.JWKS_LOAD_FAILED_CACHED_CONTENT` (117), `WARN.JWKS_LOAD_FAILED_NO_CACHE` (118), `WARN.BACKGROUND_REFRESH_IO_ERROR` (137), `WARN.BACKGROUND_REFRESH_KEY_ERROR` (139). Remnants of earlier refactoring.
|Remove all 12 unused constants. Unblocks identifier renumbering before 1.0.

|M-12
|`ClaimValue:114-127`
|**Magic null sentinel.** `isPresent()` and `isEmpty()` use `originalString == null` as sentinel for missing claim. `ClaimValue.forPlainString(null)` creates a "present but null" object — opposite of `Optional` semantics callers expect. Well-documented and consistently applied, but confusing API.
|Use `Optional<ClaimValue>` in claims maps instead of sentinel objects with null internals. At minimum, document the behavior.

|M-13
|`MinimalTokenContent` interface
|**Unsealed base interface.** `MinimalTokenContent` is not sealed, allowing third-party implementations that could bypass validation guarantees.
|Seal to `permits BaseTokenContent, RefreshTokenContent`.

|M-14
|`RefreshTokenContent:74-75`
|**Leaking internal claims.** `@Getter` on `claims` generates `getClaims()` not part of any interface contract. Exposes unvalidated data on a type that intentionally avoids `TokenContent`. Javadoc documents intent for JWT-formatted refresh tokens, but no production caller uses it.
|Remove `@Getter` on `claims` to keep it internal.

|M-15
|`HttpJwksLoaderConfig:593-601`
|**Executor thread leak.** Auto-created `ScheduledExecutorService` is never shut down — `close()` only cancels the `ScheduledFuture`. Daemon threads mitigate JVM shutdown impact but the executor pool is never properly closed.
|Track ownership with an `ownsExecutor` flag. Shut down the service in `close()` when owned.

|M-16
|`ClientIpExtractor:44-58`
|**Untrusted proxy headers.** `X-Forwarded-For` trusted at face value without documenting the trust model. Deployments without a reverse proxy get spoofable IPs in access logs.
|Add Javadoc warning about the trust model. Consider a config option for trusted headers.

|M-17
|`BearerTokenProducer:119-124`
|**Field injection for config.** `tokenHeader` and `tokenCookieName` use `@ConfigProperty` field injection on `@ApplicationScoped` bean, inconsistent with the rest of the class which uses constructor injection.
|Move to constructor injection parameters for consistency.
|===

== Low (Cleanup)

[cols="1,3,4,3"]
|===
|ID |Location |Issue |Recommendation

|L-1
|`AccessTokenContent:68`
|**Unused LOGGER field.** Declared but never referenced — all logging delegated elsewhere.
|Remove.

|L-2
|`AccessTokenContent:94-98`
|**Unused `getTokenId()`.** Zero callers in production or tests.
|Remove before 1.0.

|L-3
|`DecodedJwt:139-185`
|**Duplicated precondition.** `getSignatureAsDecodedBytes()` and `getDataToVerify()` both duplicate the same `parts == null \|\| parts.length != 3` check.
|Extract a private `validateParts()` or validate in the compact constructor.

|L-4
|`DecodedJwt:259-304`
|**Hand-rolled builder on a record.** Records are not designed for builder patterns. All fields are always set together.
|Replace with a static factory `DecodedJwt.of(...)`.

|L-5
|`SignatureAlgorithmPreferences:60-66`
|**Constructor accepts rejected algorithms.** Caller could pass `List.of("none")` — constructor does not validate against the rejected list.
|Add constructor-time validation rejecting any algorithm in `REJECTED_ALGORITHMS`.

|L-6
|`DpopProofValidator:339-349`
|**htu/htm validation silently skipped.** When DPoP proof is present but `requestUri`/`requestMethod` are missing, validation is skipped with only a WARN log. A proof for a different endpoint is accepted.
|Reject the proof rather than skipping validation when a DPoP proof is present.

|L-7
|`DpopProofValidator:435-437`
|**`SignatureTemplateManager` created per call.** New instance on every DPoP validation causes unnecessary object churn.
|Move creation to constructor.

|L-8
|`JwksParser:50-58`
|**Unused 3-arg constructor.** `@RequiredArgsConstructor` generates it but all callers use the 2-arg constructor. `dslJson` is derived from `parserConfig`.
|Remove `@RequiredArgsConstructor`, keep only the 2-arg constructor.

|L-9
|`Jwks:53-73`
|**Unused convenience methods.** `empty()`, `isEmpty()`, `size()` not called from production code.
|Remove `empty()` and `size()`. Keep `isEmpty()` if useful for future validation.

|L-10
|`MapRepresentation:114-116`
|**Unused `empty()` in production.** Never called in production code. Used extensively in tests (30+ usages) as test utility.
|Remove if test utility is not considered part of public API contract.

|L-11
|`WellKnownConfigurationConverter:148-156`
|**Fragile heuristic for security violations.** Uses substring matching (`contains("buffer")`, `contains("limit")`) on IOException messages. Any exception with "limit" in the message is misclassified.
|Check for specific DSL-JSON exception types or match more specific strings.

|L-12
|`JwksParser:159-170`
|**Double UTF-8 encoding.** `validateContentSize` encodes to bytes for size check, then `parse` encodes again. Wastes ~2x payload size for large inputs.
|Encode once and pass the byte array through.

|L-13
|`KeyProcessor:66-78`
|**Duplicate kty validation.** `processKey` repeats the same null/empty check that `validateKeyParameters` already performs.
|Remove the duplicate check in `processKey`.

|L-14
|`JWKSKeyLoader:280-282`
|**Test-only method in production.** `getKeyInfoMap()` only called from tests. Leaks internal state.
|Remove. Tests should use the public `getKeyInfo(kid)` method.

|L-15
|`WellKnownResult:47-75`
|**Mutable class with public fields.** Unlike all other JSON model classes (records), `WellKnownResult` has directly assignable public fields. Deliberate trade-off for DSL-JSON `@CompiledJson` performance, but DSL-JSON does support records.
|Convert to a record for consistency and immutability.

|L-16
|`CollectionClaimHandler:95-96`
|**Misleading `@SuppressWarnings`.** Comment says "implementation already uses a Set" but `asList` is `List<String>`. Only `ScopeMapper` produces a `TreeSet`-backed list; other mappers produce `ArrayList`.
|Either store a `Set` internally or remove the misleading comment.

|L-17
|`OAuthSheriffRecorder:33-35`
|**Empty recorder class.** Never referenced from `OAuthSheriffProcessor`. Dead code.
|Remove.

|L-18
|`AccessLogFilterConfigResolver:85`
|**No error handling on Integer.parseInt.** Non-integer config values cause cryptic `NumberFormatException`.
|Wrap in try-catch with a clear error message including the property key.

|L-19
|`KeycloakDefaultGroupsMapper:66`, `KeycloakDefaultRolesMapper:69`
|**Eager toString in debug log.** `mapRepresentation.toString()` called on every invocation even when debug is disabled.
|Guard with `if (LOGGER.isDebugEnabled())` or use lazy evaluation without `.toString()`.

=== Top Priorities for Pre-1.0

1. **H-1**: Fix TOCTOU race in `JWKSKeyLoader.initJWKSLoader` — use `AtomicBoolean.compareAndSet`
2. **H-2**: Add catch-all in `HttpJwksLoader` background refresh — prevent silent task death
3. **H-4**: Resolve `@BearerAuth` annotation merging — documentation vs. behavior mismatch is a security gap
4. **M-3**: Bound the `ClaimName` cache — attacker-controlled memory growth
5. **M-5**: Harden `JweDecryptor` JSON parsing — use shared `ParserConfig` limits
6. **M-4**: Fix `AccessTokenCache` key collisions — use full token string or crypto hash
7. **H-3, M-1, M-2, M-9, M-10, M-11**: Remove dead API surface — clean public contract before 1.0 locks it
