= Code Findings: Compatibility vs. Correct Design
:toc: left
:toclevels: 3
:sectnums:

== Overview

Semantic analysis of all production code asking one question per artifact:
_"Does this exist because it is the right design, or because we are afraid to break something?"_

Maturity: pre-1.0 — Breaking changes are free. No deprecation needed. Recommend removing/fixing everything.

Date: 2026-04-04

Modules analyzed: `oauth-sheriff-core` (107 files), `oauth-sheriff-quarkus` (42 files), `oauth-sheriff-quarkus-deployment` (2 files).

== Critical (Security Impact)

No critical security findings. The codebase fails closed: issuers default to disabled, audience validation is enforced, the bearer token interceptor rejects on any validation failure, and DPoP validation throws when request URI/method are unavailable.

== High (API / Design)

[cols="1,3,4,3"]
|===
|ID |Location |Issue |Recommendation

|H-1
|`IssuerConfig.java` (class-level `@Getter`)
|Lombok `@Getter` on the class makes ALL fields publicly accessible, including internal implementation details like `jwksLoader` that shouldn't be part of the library's public API. External consumers (quarkus health check, DevUI) do call `getJwksLoader()`, but so would any library user.
|Use `@Getter(AccessLevel.PACKAGE)` on internal fields like `jwksLoader` and selectively expose only the intended public API fields.

|H-2
|`OAuthSheriffDevUIRuntimeService.java:61`
|`ISSUER_CONFIG_DEFAULT_CLOCK_SKEW_SECONDS = 60` is a local copy of the core builder default. If the core default changes, this value silently diverges.
|Expose the default as a public constant from `IssuerConfig` and reference it, or remove the fallback (the `issuerConfigs` list is guaranteed non-empty after init).

|===

== Low (Cleanup)

[cols="1,3,4,3"]
|===
|ID |Location |Issue |Recommendation

|L-1
|`TokenHeaderValidator.java:174-178`
|Dead code: `kidOpt.isPresent()` check within the `kid.isEmpty()` branch. We entered this branch because `kid.isEmpty()` was true at line 162, and `kid` delegates to `header.getKid()` — same value as `kidOpt`. The block at lines 174-178 can never execute.
|Remove the `kidOpt` check block (lines 174-178) from the error message construction.

|L-2
|`DpopProofValidator.java:83`
|`dslJson` field creates a new unconfigured `DslJson<>()` instance without security limits (`limitStringBuffer`, `limitDigitsBuffer`), separate from the `ParserConfig`-configured instance used by `NonValidatingJwtParser`. The 8KB size check provides primary protection, but the DslJson limits add defense-in-depth.
|Pass the `DslJson` instance from `ParserConfig` or create one with the same security limits.

|L-3
|`ClaimValue.java:99-105`
|Package-private constructor allows creating inconsistent instances (e.g., `ClaimValueType.STRING` with non-null `dateTime` and non-empty `asList`). Factory methods enforce correctness, but the constructor is accessible to internal code.
|Add validation in the constructor to ensure the `type` matches the populated fields, or make it private.

|L-4
|`JwtMetricsCollector.java:192`
|Unnecessary null-check on constructor-injected `securityEventCounter` in `clear()`. CDI would fail at startup if unsatisfied. The null-check hides potential initialization bugs.
|Remove the null-check. If the field is ever null, the NPE is the correct signal.

|L-5
|`OAuthSheriffDevUIRuntimeService.java:158-159`
|Hardcoded default values for HTTP timeouts (`"per-issuer (default: 10)"`) could diverge from actual `HttpJwksLoaderConfig` defaults.
|Extract defaults from the config class or mark these as approximate.

|L-6
|`BearerTokenProducer.java:214-216`
|Invalid `cui-rewrite:disable CuiLogRecordPatternRecipe` suppression with nonsensical comment ("cross-module boundary log"). The quarkus module has its own `OAuthSheriffQuarkusLogMessages` class with WARN records — there is no reason to suppress this.
|Create a WARN LogRecord (identifier 129 in range 121-130) in `OAuthSheriffQuarkusLogMessages.WARN` and use it. Remove the suppression.

|L-7
|`BenchmarkLoggingSetup.java:69-114`
|Uses `System.err.println()` for logging during bootstrap (before JUL is configured), with a `cui-rewrite:disable CuiLoggerStandardsRecipe` suppression. Since Java 9, `System.Logger` (JEP 264) writes to stderr by default via JUL's `ConsoleHandler` and automatically picks up JUL configuration once it is set up later — making `System.err.println()` unnecessary.
|Replace `System.err.println()` with `System.Logger` (`System.getLogger(BenchmarkLoggingSetup.class.getName())`). Remove the `cui-rewrite:disable` suppression.

|L-8
|`JwtValidationEndpoint.java:101-102`, `ConcurrencySweepRunner.java:64-65,90-91,101-102`
|Unfixed OpenRewrite TODO markers left in production source code (e.g., `/*~~(TODO: WARN needs LogRecord)~~>*/`). These are diagnostic comments that should have been resolved and removed. `ConcurrencySweepRunner.java:91` also has a `%.1f` placeholder bug (should be `%s`).
|Fix the placeholder bug, add proper class-level `cui-rewrite:disable` suppressions where appropriate (benchmarking/test infrastructure), and remove all markers.

|===

== Dismissed Findings (False Positives)

The following candidates were identified during per-module analysis but dismissed during cross-module verification:

* ~~**AccessTokenContent.java:142-153** — `getEmail()` and `getPreferredUsername()` have zero production callers.~~ **Dismissed**: Intentional public API surface for library consumers (standard OIDC convenience accessors).

* ~~**AccessTokenContent.java:155-247** — `providesScopesAndDebugIfScopesAreMissing` and siblings have zero production callers.~~ **Dismissed**: Documented Design Decision as "part of the 1.0 public contract" for library consumers.

* ~~**BearerTokenResult.java:116-126** — error factory methods store unused required scopes/roles/groups.~~ **Dismissed**: Intentional design — the fields are part of the result model for library consumers who may inspect them.

* ~~**ClaimName.java:130-134** — `ClaimName.TYPE` has zero production callers and is semantically incorrect (header parameter in payload enum).~~ **Dismissed**: `ClaimName.TYPE` models a custom payload `typ` claim (documented as "implementation-specific"), not the JWT header `typ`. It participates in the `CLAIM_NAME_LOOKUP` map — `forName("typ")` resolves it at runtime without needing direct constant references.

* ~~**SecurityEventCounter.java:124-125** — `EventType.logRecord` field stored but never read.~~ **Dismissed**: Documented design decision. The field enforces compile-time pairing between each `EventType` and its corresponding `LogRecord`, preventing addition of event types without considering their log message.

* ~~**BearerTokenProducer.java:204-211** — CONSTRAINT_VIOLATION result drops the validated `accessTokenContent`.~~ **Dismissed**: This is the intended design. The CONSTRAINT_VIOLATION path means the token was valid but the user lacks required scopes/roles/groups. The interceptor rejects the request; the token content is not needed in the error result.

* ~~**ClientIpExtractor.java** — case-sensitive proxy header matching.~~ **Dismissed**: Quarkus/RESTEasy Reactive's `ContainerRequestContext.getHeaders()` returns a case-insensitive `MultivaluedMap` backed by Vert.x headers.

* ~~**BearerTokenProducer.java:234-244** — `resolveRequestContext()` silently returns `RequestContext(null, null)` on failure, potentially bypassing DPoP htu/htm validation.~~ **Dismissed**: The core `DpopProofValidator.validateHtuHtm()` at line 348-350 throws `TokenValidationException` when request URI/method are null. This is correct fail-closed behavior.

* ~~**CustomAccessLogFilter.java:149** — uses `java.nio.file.PathMatcher` for URL paths.~~ **Dismissed**: This is a logging feature (not security), Quarkus runs on Linux/macOS, and the `FileSystems.getDefault()` path separator matches URL path separators on these platforms.

* ~~**ClaimName.UPN** — zero production callers in core.~~ **Dismissed**: Used in quarkus module's `JsonWebTokenAdapter` for MP-JWT principal name resolution.

* ~~**StringSplitterMapper** — zero production callers in core.~~ **Dismissed**: Documented public SPI; referenced in quarkus deployment processor for native image reflection registration.

* ~~**IdTokenContent.getDisplayName()/getEmail()** — zero production callers in core.~~ **Dismissed**: `getEmail()` is called in `oauth-sheriff-quarkus-integration-tests` endpoint (`JwtValidationEndpoint.java:429`). Standard OIDC convenience accessors for the public API.

* ~~**SecurityEventCounter.java C-1** — `ALGORITHM_REJECTED` log record has no corresponding EventType.~~ **Dismissed**: The caller `TokenHeaderValidator` increments `UNSUPPORTED_ALGORITHM` for all rejected algorithms, which covers both unsupported and explicitly-rejected (HMAC/none) cases. The specific `ALGORITHM_REJECTED` log record provides diagnostic detail while the security event counter tracks the broader category.

* ~~**SecurityEventCounter.java L-1** — success EventTypes with null LogRecord/Category could cause NPE in TokenValidationException.~~ **Dismissed**: Design Decision comment documents that success events are metric counters only (DEBUG level) and are never used with `TokenValidationException`. The `getCategory()` method is null-safe by returning null for success events.

* ~~**BearerTokenProducer.java H-4** — `getAccessTokenContent()` and `produceJsonWebToken()` perform duplicate validation per request.~~ **Dismissed**: CDI `@RequestScoped` proxy caching prevents duplicate validation when both `JsonWebToken` and `Principal` are injected.
