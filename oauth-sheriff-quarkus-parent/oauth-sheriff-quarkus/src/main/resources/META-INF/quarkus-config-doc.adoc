= OAuth Sheriff Quarkus Extension Configuration Reference
:toc-title: Table of Contents
:sectnums:
:source-highlighter: highlight.js

All `sheriff.oauth.*` properties for the Quarkus extension. For the core library builder API, see xref:../../../../../../oauth-sheriff-core/doc/api-reference.adoc[Core API Reference].

== Minimal Configuration

[source,properties]
----
# OpenID Connect Discovery (recommended) — issuer, JWKS, and algorithms auto-discovered
sheriff.oauth.issuers.keycloak.jwks.http.well-known-url=https://keycloak.example.com/realms/master/.well-known/openid-configuration
----

== Issuer Configuration

Properties under `sheriff.oauth.issuers.<name>.*` — one block per issuer.

[cols="2,1,1,3", options="header"]
|===
|Property |Type |Default |Description
|`enabled` |boolean |`true` |Enable/disable this issuer
|`issuer-identifier` |String |— |Matched against `iss` claim. Auto-discovered when using well-known.
|`expected-audience` |String (csv) |— |Expected `aud` claim values
|`expected-client-id` |String (csv) |— |Expected `azp` / `client_id` claim values
|`algorithm-preferences` |String (csv) |— |Allowed signature algorithms (default: all supported)
|`claim-sub-optional` |boolean |`false` |Allow missing `sub` claim
|`expected-token-type` |String |— |Expected `typ` header (e.g. `at+jwt` per RFC 9068)
|===
