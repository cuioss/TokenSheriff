== Methodology

An ablation study decomposes the full JWT avg latency (2.30ms at 50 connections) by progressively adding processing layers. Each variant adds exactly one component, isolating its cost by comparison with the previous variant.

[cols="1,2,3,3", options="header"]
|===
|Variant
|Path
|What it does
|What it isolates

|Health (existing)
|`GET /q/health/live`
|SmallRye Health, minimal JSON
|Shared infrastructure baseline

|**B: JAX-RS Baseline**
|`POST /mock-jwt/baseline`
|CDI-injected class, pre-built response
|JAX-RS POST routing + Jackson (B minus Health)

|**C: Header Only**
|`POST /mock-jwt/header-only`
|Calls `resolveHeaderMap()`, pre-built response
|Vert.x context resolution (C minus B)

|Mock JWT (existing)
|`POST /mock-jwt/validate`
|resolveHeaderMap() + header parsing + HashMap
|Authorization parsing (Mock minus C)

|**D: Direct Validation**
|`POST /mock-jwt/direct-validation`
|Calls `createAccessToken()` directly, no CDI proxy
|JWT validation under concurrency (D minus Mock)

|JWT (existing)
|`POST /jwt/validate`
|Full JWT validation via CDI `Instance.get()`
|CDI proxy overhead (JWT minus D)
|===

All variants use POST with Authorization header (same WRK Lua script) so differences are purely server-side.

NOTE: A historical Variant A (bare JAX-RS without CDI) confirmed that `@ApplicationScoped` injection has zero per-request cost. It was removed after serving its purpose.

== Key Findings

* **CPU saturation from RSA under concurrency is the sole significant cost** at 46.1% (1,060µs) of request time. The core library validates in 48µs at 1 thread (link:Analysis-03.2026-Micro.adoc[micro-benchmark]). A JMH concurrency sweep shows this scales with thread-to-core oversubscription (556µs at 50 threads on HotSpot JVM, 10 cores). See link:Analysis-03.2026-Integration.adoc#_root_cause_cpu_saturation_from_rsa_under_concurrency[root cause analysis].
* **Shared infrastructure + JAX-RS routing** account for 51.3% — expected overhead for HTTP-based microservices.
* **All other layers are noise**: Vert.x resolution (0.06ms), authorization parsing (~0ms), and CDI proxy (~0ms) are individually negligible.
