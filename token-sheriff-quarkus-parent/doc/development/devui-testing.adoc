== Overview

DevUI components are tested at two levels:

* **Unit tests (Jest)** -- Individual Lit components in isolation with mocked JSON-RPC responses
* **E2E tests (Playwright)** -- Full DevUI integration against a running Quarkus dev instance

== Unit Tests (Jest)

[source,bash]
----
npm test
npm run test:watch
npm run test:coverage
npm run test:ci
----

=== Test Structure

----
token-sheriff-validation-quarkus-deployment/src/test/js/
├── setup/              # Jest configuration and setup
├── mocks/              # Mock implementations (Lit, DevUI, directives)
└── components/         # Component-specific test files
----

=== Component Test Files

* `qwc-jwt-status-config.test.js` -- Status and configuration viewer
* `qwc-jwt-debugger.test.js` -- Token validation tool
* `simple-component.test.js` -- Basic component rendering

=== Custom Test Utilities

* `toHaveRenderedContent()` -- Verify component rendering
* `toHaveShadowClass()` -- Check shadow DOM classes
* `toBeDefinedAsCustomElement()` -- Validate custom element registration
* `waitForComponentUpdate()` -- Handle async component updates
* `createTestComponent()` / `cleanupTestComponents()` -- Component lifecycle helpers

== E2E Tests (Playwright)

=== Test Structure

----
e-2-e-playwright/
├── tests/
│   ├── self-devui-accessible.spec.js          # Environment gate-keeper tests
│   ├── 01-verify-status-config.spec.js        # Status & Config page
│   ├── 03-verify-token-debugger.spec.js       # Token Debugger page

│   └── test-logger.js                         # Structured test logging
└── fixtures/
    └── test-fixtures.js                       # Shared test fixtures
----

=== Running E2E Tests

[source,bash]
----
# Run from token-sheriff-quarkus-parent/ directory
docker compose up -d
./mvnw -f token-sheriff-quarkus-integration-tests/pom.xml quarkus:dev
cd e-2-e-playwright && npx playwright test
----

=== Navigation Strategy

Quarkus DevUI is a SPA (Vaadin Router + Lit web components) where extension sub-page routes are registered lazily. Tests use click-based navigation through the extension card, traversing shadow DOM to reach anchors inside `qwc-extension-link` elements.

=== Test Coverage

* **Self-tests (6)**: Environment validation
* **Functional tests (20)**: Status & Config data rendering (13), Token Debugger interactive features (7)
